What happens if I delete a when case in notification settings?
There’s no side effects. You will have to click “Save notification” under all settings to save all modifications.

What happens if I delete a notification?
There’s no side effects. You will just lose that particular notification and it will be deleted permanently. Also, the logs will be deleted and you will not be able to see them.

What are when cases & how do they work?
When cases are a set of options that define situations when notifications should be fired. Each when case has a selected action type:

What are the options for an action type: variable in ranges?
  1. Variable - you have to select an existing variable to track from data sources
  2. Range change - you have to select one of the range change events
  3. Stay in range - defines how long a value has to stay in a new range before it triggers this when case
  4. Wait before firing again - you can set if when cases should be fired several times and how long to wait before firing again.
  5. Allow this to fire at any time - switch indicating if other “when” case must be fired on the same variable to fire again

What are the options for an action type: getting and not getting data?
  1. Getting or not getting for specified minutes. You can select how long to wait before firing actions. The minimum amount is 5 minutes because it’s ok if your server is running but a network failure resulted in logs not being delivered to Trafikito. However, if it happens for more than 5 minutes then it’s an issue not just a random event.
  2. Wait before firing again - you can set if when cases should be fired several times and how long to wait before firing again.

What are possible range changes to detect?
You can detect any possible value of a selected variable range change. You can select these ranges in the when case options while editing notifications.
